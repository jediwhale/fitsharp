Writing Tests
We can write fitSharp tests in two formats:
  • HTML table format: This is the classic format that you will see described in the Fit literature. We write a test page directly in HTML, or with wiki markup that translates to HTML. fitSharp processes the tables in the test page and ignores the rest of the page. So we can include additional information anywhere else on the page. Html Comments can be included. Tidy Html is handled.
  • Plain Text Format : This is a new format introduced by fitSharp. We use special delimiters to indicate the text that fitSharp processes and we can also include additional information on the page.
We can mix tests in either format within a test suite, but an individual test page must be in one format.
