Define Fixture
The Define fixture lets us create reusable test procedures that can be run by other fixtures.

do

We declare the procedure in the first row of the Define table and write the procedure body as the rest of the table. We can also declare parameters in the first row. We use the Do Fixture style, where alternating cells make up the procedure name and the parameter names.

We invoke the procedure by using the procedure name in any place where we'd use a fixture or domain method name. The test results will include an expandable section showing the details of the procedure execution.

when test is
definechangename
setnameBob

definecheck fieldfieldnameisexpected
checkfieldnameexpected

do
withnewsample domain
changename
check fieldnameisBob
then result is
definechangename
setnameBob

definecheck fieldfieldnameisexpected
checkfieldnameexpected

do
withnewsample domain
changename⇓

setnameBob
check field⇓

checknameBob
nameisBob

We can define a multi-table procedure by placing nested tables in the first cell of the second row of the Define table.

checknamenewname
