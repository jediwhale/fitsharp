Folder Runner With Tags
This is a description of the tagging feature of Folder Runner. Read more detailed information about tagging in Tagging Test Pages.
 
namefile systemwithnewfile system

The test files used in the following examples are from Sample Input Files and Sample Output Files.

withfile system
make empty foldertestoutput
alias folderdocument\fitSharp\Fit\SampleOutputFilesasout

Folder Runner, with -t followed by tagList, executes all pages with given tags.

withnewfolder runner story
run-c,document\fitSharp\Fit\SampleInputFiles\SampleConfig.xml,-t,SampleTag

Test result counts are accumulated for all files processed. Note that the suite setup and suite teardown are included, which makes the number of assertions larger than expected.

checkresults3 right, 0 wrong, 0 ignored, 0 exceptions

Checks whether the result html generated is as expected. We are concerned about SampleSimpleTest.html because that is the only page which has been tagged.

withfile system
ensurefiletestoutput\SampleSimpleTest.htmlsame asout\SampleSimpleTest.html

Wildcard matching

withfile system
make empty foldertestoutput

Wildcards are evaluated in tag filters.

withnewfolder runner story
run-c,document\fitSharp\Fit\SampleInputFiles\SampleConfig.xml,-t,Samp*T?g

The wildcard filter should have matched the tagged SampleSimpleTest.html page again.

withfile system
ensurefiletestoutput\SampleSimpleTest.htmlsame asout\SampleSimpleTest.html
